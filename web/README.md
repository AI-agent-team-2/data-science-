# Site Bot - Веб-интерфейс

Веб-альтернатива Telegram-боту. Тот же AI-агент (`run_agent`), но доступный
через браузер. Страница с чатом общается с FastAPI-сервером, который вызывает
ту же логику, что и Telegram-бот.

---

## Архитектура

```
+-------------------+         +------------------+         +----------------+
|                   |  HTTP   |                  |  вызов  |                |
|  Браузер          | ------> |  FastAPI-сервер  | ------> |  run_agent()   |
|  (index.html)     | <------ |  (api.py)        | <------ |  RAG / LLM /   |
|  JS + fetch()     |  JSON   |  uvicorn         |  str    |  Web Search    |
+-------------------+         +------------------+         +----------------+
```

Браузер отправляет POST-запрос с текстом сообщения и `session_id`.
Сервер передает их в `run_agent()` — ту же функцию, которую использует
Telegram-бот. Ответ возвращается как JSON и отображается в чате.

---

## Предварительные требования

1. **Python 3.10+** установлен
2. **`.env` файл** в корне проекта настроен (см. `.env.example`)
3. **ChromaDB проиндексирована** (если используете RAG — запустите `python -m app.rag.ingest`)

## Установка и запуск

### Шаг 1. Установите зависимости

Из корня проекта:

```bash
pip install -r requirements.txt
```

Это установит все зависимости, включая `fastapi` и `uvicorn`.

### Шаг 2. Запустите веб-сервер

```bash
python -m uvicorn web.api:app --reload --port 8000
```

- `web.api:app` — путь к FastAPI-приложению (файл `web/api.py`, переменная `app`)
- `--reload` — автоматическая перезагрузка при изменении кода (удобно при разработке)
- `--port 8000` — порт сервера

### Шаг 3. Откройте в браузере

```
http://localhost:8000
```

Вы увидите страницу чата. Введите вопрос и нажмите «Отправить».

---

## Как это работает (пошагово)

1. Пользователь вводит вопрос в текстовое поле и нажимает «Отправить».
2. JavaScript берет текст и `session_id` (хранится в `localStorage` браузера).
3. JS отправляет POST-запрос на `/api/chat` с JSON-телом:
   ```json
   {"message": "Какие смесители ONDO есть?", "session_id": "abc-123"}
   ```
4. FastAPI получает запрос, валидирует через Pydantic-модель `ChatRequest`.
5. Вызывается `run_agent(message, user_id=session_id)` — та же функция,
   что использует Telegram-бот.
6. `run_agent()` внутри:
   - Классифицирует запрос (RAG / поиск по артикулу / веб-поиск)
   - Загружает историю диалога из SQLite
   - Собирает контекст из нужных источников
   - Отправляет промпт в LLM и получает ответ
   - Сохраняет обмен в историю
7. FastAPI возвращает JSON: `{"reply": "...", "session_id": "abc-123"}`
8. JS получает ответ и добавляет «пузырь» сообщения бота в чат.

## Справочник API

### `GET /api/health` — Проверка работоспособности

```bash
curl http://localhost:8000/api/health
```

Ответ:

```json
{"status": "ok"}
```

### `POST /api/chat` — Отправка сообщения

```bash
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Что такое смеситель?", "session_id": "test-123"}'
```

Запрос:

| Поле | Тип | Описание |
|------|-----|----------|
| `message` | string (1-2000) | Текст сообщения пользователя |
| `session_id` | string (1-100) | Идентификатор сессии |

Ответ:

```json
{"reply": "Смеситель — это...", "session_id": "test-123"}
```

### `POST /api/clear` — Очистка истории

```bash
curl -X POST http://localhost:8000/api/clear \
  -H "Content-Type: application/json" \
  -d '{"session_id": "test-123"}'
```

Ответ:

```json
{"status": "ok"}
```

## Ключевые концепции

### REST API

REST (Representational State Transfer) — архитектурный стиль для веб-сервисов.
Клиент (браузер) и сервер общаются через HTTP-запросы. Каждый эндпоинт — это
URL + метод (GET, POST, PUT, DELETE). В нашем случае:
- `POST /api/chat` — отправить сообщение (создание ресурса — ответа)
- `GET /api/health` — получить состояние (чтение)

### CORS

Cross-Origin Resource Sharing — механизм безопасности браузера. Без CORS
браузер запретит JS-коду обращаться к API на другом домене/порту.
В `api.py` мы включаем `CORSMiddleware` с `allow_origins=["*"]` — разрешаем
запросы отовсюду (подходит для локальной разработки, в продакшене стоит
указать конкретный домен).

### session_id вместо Telegram user_id

В Telegram у каждого пользователя есть уникальный числовой `user_id`. В вебе
такого нет, поэтому мы генерируем UUID в браузере (`crypto.randomUUID()`) и
сохраняем его в `localStorage`. Этот UUID играет роль `user_id` — по нему
`run_agent()` загружает и сохраняет историю диалога.

### Почему `def`, а не `async def`

Функция `run_agent()` — синхронная: она делает блокирующие HTTP-вызовы к LLM
и обращения к SQLite. Если объявить эндпоинт как `async def`, FastAPI будет
выполнять его в основном asyncio-потоке, и блокирующий вызов «заморозит»
весь сервер. При объявлении как обычный `def` FastAPI автоматически выполняет
функцию в пуле потоков (`ThreadPoolExecutor`), что корректно для нашего случая.

## Одновременная работа с Telegram-ботом

Веб-сервер и Telegram-бот могут работать параллельно — они используют одну
и ту же базу SQLite для истории, и SQLite корректно обрабатывает конкурентные
чтения. Для учебного проекта с небольшим числом пользователей этого достаточно.

Запуск обоих:

```bash
# Терминал 1 — Telegram-бот
python -m app.bot.telegram_bot

# Терминал 2 — Веб-сервер
python -m uvicorn web.api:app --reload --port 8000
```

## Задания для самостоятельной работы

### 1. Добавьте эндпоинт `/api/history`

Создайте GET-эндпоинт, который возвращает историю диалога для данного
`session_id`. Используйте функцию `load_messages()` из `app/history_store.py`.

Подсказка: параметр `session_id` можно передать как query-параметр:
`GET /api/history?session_id=abc-123`

### 2. Рендеринг Markdown в ответах бота

Бот иногда возвращает текст с Markdown-разметкой (списки, жирный текст).
Подключите библиотеку [marked](https://github.com/markedjs/marked) через CDN
и рендерите ответы бота как HTML вместо plain text.

Помните о безопасности: используйте sanitizer (например, DOMPurify) при
рендеринге HTML из внешних данных.

### 3. Индикатор «бот печатает»

Замените текущую анимацию точек на более информативный индикатор: показывайте
текст «SAN Bot печатает...» с анимацией, который исчезает при получении ответа.

### 4. Деплой на VPS

Настройте запуск веб-сервера через systemd на VPS (аналогично Telegram-боту,
см. `DEPLOY_VPS.md`). Используйте nginx как reverse proxy для обслуживания
статических файлов и проксирования API-запросов.
