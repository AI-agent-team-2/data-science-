## [2.0.0] - 2026-03-21

### 🚀 Добавлено (Added)

#### Веб-поиск
- **Кэширование результатов** — добавлен in-memory кэш для повторяющихся запросов (до 100 записей, TTL 24 часа)
- **Фильтрация по сантехнике** — функция `_is_sanitary_relevant` отсеивает результаты, не содержащие ключевых слов (унитаз, смеситель, кран, труба, сантехника)
- **Улучшение поисковых запросов** — `enhance_search_query` автоматически расширяет запросы:
  - «новинки 2026» → «новинки сантехники 2026 каталог»
  - «сколько стоит» → «сантехника цена купить»
  - «где купить» → «сантехника интернет магазин»

#### Telegram бот
- **Новые команды**:
  - `/help` — справка с примерами запросов
  - `/clear` — очистка истории диалога пользователя
- **Инлайн-кнопки** — интерактивные кнопки для выбора типа поиска

#### RAG (поиск по базе знаний)
- **Гибридный поиск** — `hybrid_search` комбинирует векторный поиск с бустингом по ключевым словам
- **Извлечение ключевых слов** — `_extract_keywords` выделяет значимые слова из запроса

#### Мониторинг (новый модуль `app/monitoring.py`)
- **Декоратор `track_time`** — замер времени выполнения функций
- **Сбор метрик** — время, количество вызовов
- **Статистика** — `get_stats()` возвращает среднее, min, max, количество
- **Статистика за сегодня** — `get_today_stats()`

#### История диалогов
- **Очистка истории** — `clear_history()`
- **Статистика истории** — `get_history_stats()`

#### Конфигурация
- Новые параметры в `.env`:
  - `web_cache_enabled` — включение кэша
  - `web_cache_ttl_hours` — время жизни кэша
  - `web_search_max_results` — лимит результатов
  - `enable_web_search`, `enable_rag`, `enable_product_lookup` — переключение инструментов

### 🐛 Исправлено (Fixed)

#### Неизвестные slash-команды
- **Проблема:** Команды `/status`, `/id` и другие неизвестные команды уходили в LLM как обычный текст
- **Исправление:** 
  - Добавлены команды `/status` (статус бота, версия, состояние RAG) и `/id` (chat_id, user_id)
  - Добавлен перехватчик для всех неизвестных slash-команд с ответом "Неизвестная команда, используйте /help"
  - Обновлен `/help` с новыми командами
  
#### Роутинг SKU-first на коротких запросах
- **Проблема:** Запросы короче 3 слов всегда направлялись в web_search, даже если содержали SKU (артикул)
- **Исправление:** Добавлена проверка на SKU перед обработкой коротких запросов; SKU-запросы теперь корректно идут в product_lookup

#### Блок "Источники" при ответах из RAG/LOOKUP
- **Проблема:** Блок источников добавлялся, если `prefer_web = True`, даже если фактически ответ был из RAG или LOOKUP
- **Исправление:** Добавлен флаг `web_actually_used`, блок источников добавляется только при реальном использовании веб-поиска

#### Применение настроек из конфига
- **Проблема:** Переменные `ENABLE_WEB_SEARCH`, `WEB_CACHE_ENABLED`, `WEB_CACHE_TTL_HOURS`, `WEB_SEARCH_MAX_RESULTS` не применялись в рантайме
- **Исправление:** `web_search.py` теперь использует настройки из `app.config.settings`

#### TTL-очистка истории
- **Проблема:** Строковое сравнение дат в SQLite могло давать ошибки на границе дат
- **Исправление:** Использованы SQLite datetime функции (`datetime('now', '-N days')`) для корректной фильтрации

#### Документация .env.example
- **Проблема:** Новые переменные из версии 2.0 отсутствовали в `.env.example` и README
- **Исправление:** Добавлены все недостающие переменные с описанием

### 🔧 Изменено (Changed)

- **Финальный промпт** — добавлена инструкция: «Если вопрос только про сантехнику — отвечай только про сантехнику»
- **Fallback цепочка** — теперь используется полноценная система последовательных попыток:
  - `prefer_web`: web → rag → lookup → уточнение
  - `prefer_lookup`: lookup → rag → уточнение
  - обычный запрос: rag → lookup → web → уточнение
- **Проверка полезности результатов** — улучшены критерии для всех трех инструментов
