# Changelog

Все заметные изменения проекта фиксируются в этом файле.

## [2.0.1] - 2026-03-23

### Changed
- Проведен рефакторинг:
  - единый стиль и повышение читаемости кода (PEP-8);
  - расширенная типизация (`type hints`, `TypedDict`, `TypeAlias`, `dataclass`);
  - декомпозиция крупных функций на логические блоки;
  - унификация JSON-ответов инструментов;
  - добавлены/уточнены docstring.
- Обновлена инфраструктурная часть:
  - улучшена обработка ошибок и логирование в `run_agent`, `history_store`, `monitoring`, `rag` и `tools`;
  - упорядочена конфигурация в `app/config.py` с безопасным парсингом env-переменных;
  - унифицированы скрипты `scripts/retrieval_eval.py` и `scripts/web_search_smoke.py`.
- Обновлена документация:
  - `README.md` синхронизирован с фактическим поведением системы;
  - уточнено описание контроля домена ответов (через системный prompt).

### Fixed
- Исправлены несоответствия между документацией и кодом:
  - кэш `web_search` документирован как файловый (`.web_cache`), а не in-memory;
  - удалены некорректные формулировки про «защиту от prompt injection» как отдельный механизм;
  - починен обрезанный/незавершенный хвост предыдущего `CHANGELOG.md`.

## [2.0.0] - 2026-03-21

### Added
- Веб-поиск с поддержкой Tavily/DuckDuckGo.
- Кэширование результатов веб-поиска.
- Фильтрация web-результатов по сантехнической релевантности.
- Улучшение поисковых web-запросов (`enhance_search_query`).
- Команды Telegram-бота: `/help`, `/clear`, `/status`, `/id`.
- Инлайн-кнопки в Telegram-боте.
- Гибридный retrieval в `ChromaRetriever`.
- Модуль мониторинга `app/monitoring.py`.
- История диалогов: очистка и базовая статистика.

### Fixed
- Обработка неизвестных slash-команд (чтобы не уходили в LLM как обычный текст).
- SKU-first роутинг для коротких запросов.
- Корректное добавление блока «Источники» только при реальном использовании web-поиска.
- Применение env-настроек в рантайме для web-инструмента.
- TTL-очистка истории через SQLite `datetime(...)`.

### Changed
- Улучшен финальный prompt для более строгого ответа по теме запроса.
- Выстроена цепочка fallback-источников: lookup/rag/web в зависимости от типа запроса.

## Migration notes

### Обновление до 2.x
1. Обновите код:
   ```bash
   git checkout main
   git pull origin main
   ```
2. Сверьте `.env` с `.env.example` и добавьте недостающие переменные.
3. При изменении базы знаний переиндексируйте RAG:
   ```bash
   python -m app.rag.ingest
   ```
