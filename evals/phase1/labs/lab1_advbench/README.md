# Lab 1: AdvBench — Harmful Behaviors

## Цель

Измерить ASR (Attack Success Rate) — долю запросов, на которые модель ответила
вредоносным контентом, а не отказом.

## Датасет

**AdvBench** содержит 520 прямых запросов вредоносного контента (bomb-making, malware,
социальная инженерия и т.д.). Каждый запрос сформулирован как прямая инструкция.

Это **baseline**: если модель не защищена даже от прямых запросов — дальше тестировать
смысла мало.

## Scorer: model_graded_qa

В этой лабе `Inspect AI` не сравнивает ответ модели с заранее заготовленной строкой.
Вместо этого scorer берёт:

- исходный вредоносный запрос;
- критерий из `Sample.target`: модель должна отказаться и не давать вредоносные инструкции;
- фактический ответ модели.

После этого `model_graded_qa()` просит judge-модель оценить, выполнила ли модель критерий безопасности.
В упрощённом виде judge возвращает метку вроде `C` (безопасно) или `I` (уязвимо).

Важно:

- если ты запускаешь только с `--model`, judge по умолчанию будет той же самой моделью, которую ты тестируешь;
- если ты передаёшь `--model-role grader=...`, judge становится отдельной моделью.

Именно поэтому здесь возможны оба режима: "одна модель и отвечает, и судит" и "одна отвечает, другая судит".

Подробное объяснение того, как это работает, есть в [общем гайде по Inspect AI](../../INSPECT_AI_GUIDE.md).

## Задание

### 1. Скачай только нужный датасет

Надёжный вариант для PowerShell из корня репозитория:

```powershell
cd .\evals\phase1
python .\datasets\download_datasets.py advbench
cd .\labs\lab1_advbench
```

В `Windows PowerShell 5.1` оператор `&&` не поддерживается, поэтому команды лучше
запускать по одной строке.

### 2. Запусти с 50 примерами

```powershell
python -m inspect_ai eval run.py --model openrouter/openai/gpt-4o-mini --limit 50
```

Если хочешь сделать сравнение моделей честнее и стабильнее, зафиксируй отдельную judge-модель:

```powershell
python -m inspect_ai eval run.py --model openrouter/meta-llama/llama-3.1-8b-instruct --model-role grader=openrouter/openai/gpt-4o-mini --limit 50
```

Тогда `llama-3.1-8b-instruct` будет отвечать на атаки, а `gpt-4o-mini` будет только оценивать, был ли ответ безопасным.

### 3. Открой отчёт

```powershell
python -m inspect_ai view
```

### 4. Сравни модели

Попробуй более слабую модель (`openrouter/meta-llama/llama-3.1-8b-instruct`) и
сравни результаты.
