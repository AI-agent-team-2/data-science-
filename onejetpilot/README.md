# tg_bot — Telegram-бот 

Проект содержит Telegram-бота, который общается в режиме чата, хранит историю диалога и отвечает
В контекст LLM попадают **последние 5 пар** (вопрос → ответ) + текущее сообщение. 

## Что в проекте

- **`bot_local_no_rag.py`** — версия бота для локальной LLM **без RAG** (только чат + история последних 5 пар).

- **`requirements.txt`** — зависимости.

## Установка

Создайте виртуальное окружение и установите зависимости.

### Windows (PowerShell)

```bash
python -m venv myenv
.\myenv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Переменные окружения

Проект ожидает переменные в окружении (удобно хранить в `.env`):

- **`BOT_TOKEN`** — токен Telegram-бота (через @BotFather).

## Запуск

### Локальная модель (например, Ollama)

Убедитесь, что локальный OpenAI-совместимый endpoint доступен по адресу, указанному в `bot_local.py`:

- `OPENROUTER_BASE = "http://localhost:11434/v1"`

Запуск:

```bash
python bot_local_no_rag.py
```


## Частые проблемы

- **Нет ответов/ошибки авторизации**: проверьте `BOT_TOKEN`
- **Локальная модель**: убедитесь, что сервис на `http://localhost:11434/v1` запущен и выбранная модель доступна.
